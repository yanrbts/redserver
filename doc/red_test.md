## 1. 概述

### 1.1 模块功能
pkteng（Packet Engine）数据包引擎是Red-LRM红黑隔离网关的核心组件，处理红区与黑区之间的双向流量。

### 1.2 业务流分类

| 业务流类型 | 方向 | 处理函数 |
|-----------|------|---------|
| 运维报文操作 | Red -> Black | pkt_send_ope_to_black |
| 运维报文操作 | Black -> Red | pkt_reverse_ope_to_red |
| 716_IP_JHJ控制报文操作 | Red -> Black | pkt_send_ctrl_to_black |
| 716_IP_JHJ控制报文操作 | Black -> Red | pkt_reverse_ctrl_to_red |
| 加注报文操作 | Red -> Black | pkt_send_raise_to_black |
| 加注报文操作 | Black -> Red | pkt_reverse_raise_to_red |
| 54_IP_JHJ接入控制操作 | Red -> Black | pkt_send_ctrl54_to_black |
| 54_IP_JHJ接入控制操作 | Black -> Red | pkt_reverse_ctrl54_to_red |

## 2. OPE操作数据包业务流测试

### 2.1 业务流描述
OPE（Operation）数据包用于红区客户端与黑区服务器之间的常规业务通信。

```
┌─────────┐     OPE请求      ┌─────────┐     隧道封装      ┌─────────┐
│ 红区    │ ──────────────> │  pkteng │ ──────────────> │ 黑区    │
│ 客户端  │                 │         │                 │ 服务器  │
└─────────┘                 └─────────┘                 └─────────┘
      ^                           │                           │
      │                           │  OPE响应                   │
      │  原始数据返回               │  隧道解封装                │
      └───────────────────────────┘ <───────────────────────────┘
```

### 2.2 测试用例

#### TC_OPE_001: 完整OPE双向通信
| 项目 | 内容 |
|------|------|
| **测试目标** | 验证OPE请求和响应的完整双向传输 |

| **验证点** | - 函数返回true<br>- 原始客户端收到响应数据 |

#### TC_OPE_002: OPE分片传输与重组
